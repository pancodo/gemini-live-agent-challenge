# Visual Interleave Agent — Balanced Image/Video Beat System

**Date:** 2026-03-14
**Status:** Implementation

## Problem

The documentary player is ~95% Gemini beat illustrations. Imagen 3 images are dimmed to 20% opacity as ambient background. Veo 2 videos rarely appear. For a Creative Storyteller entry, all three visual modalities should flow together naturally.

## Solution

A new **Visual Interleave Agent** (Phase 3.3) sits between Phase 3.2 (Beat Illustration) and Phase 4.0 (Narrative Visual Planner). It reads all beats from session state, assigns each beat a `visual_type`, and writes assignments back. Phase V then generates the right asset per beat.

### Beat Visual Types

| Type | Source | Generated By |
|---|---|---|
| `illustration` | Gemini interleaved TEXT+IMAGE | Phase 3.2 (already exists) |
| `cinematic` | Imagen 3 photorealistic frame | Phase V (beat-specific prompt) |
| `video` | Veo 2 short clip (~4s) | Phase V (beat-specific prompt) |

### Agent Design

**Input:** `session.state["beats"]` (dict[segment_id, list[beat_dict]])
**Output:** `session.state["beat_visual_plan"]` (dict[segment_id, list[BeatVisualAssignment]])

Single Gemini 2.0 Flash call per segment. Prompt constraints:
- Each segment MUST have at least 1 of each type (3-4 beats guarantees this)
- `video` → beats with motion/action/transformation (battles, journeys, ceremonies)
- `cinematic` → wide establishing shots, portraits, landscapes
- `illustration` → introspective, abstract, transitional moments
- Returns JSON: `[{ beat_index, visual_type, generation_prompt }]`

For `cinematic` and `video` beats, the agent writes a `generation_prompt` tailored to the beat's narration + visual bible. For `illustration` beats, `generation_prompt` is null.

### Pipeline Integration

```
Phase 3.1 → Phase 3.2 → Phase 3.3 (NEW) → Phase 4.0 → Phase IV → Phase V (MODIFIED)
```

Phase V reads `beat_visual_plan` and for each segment:
1. Skip `illustration` beats (already have Phase 3.2 image)
2. For `cinematic` beats: call Imagen 3 with `generation_prompt`
3. For `video` beats: call Veo 2 with `generation_prompt`
4. Write asset URLs back to `session.state["beats"]` with new fields

### SSE Events

New `narration_beat` fields:
- `visualType`: `'illustration' | 'cinematic' | 'video'`
- `cinematicUrl`: signed GCS URL for Imagen 3 frame (cinematic beats only)
- `videoUrl`: signed GCS URL for Veo 2 clip (video beats only)

New SSE event `beat_visual_assigned`:
- Emitted per-beat when the interleave agent assigns a visual type
- `{ type, segmentId, beatIndex, visualType }`

### Frontend Changes

**Types (`types/index.ts`):**
- Add `visualType` to `NarrationBeat` and `NarrationBeatEvent`
- New type: `BeatVisualType = 'illustration' | 'cinematic' | 'video'`

**Player Store (`playerStore.ts`):**
- `addBeat` already handles beat updates — just needs to merge new URL fields

**KenBurnsStage (`KenBurnsStage.tsx`):**
- Current beat rendering (zIndex 2) checks `currentBeatImage` only
- New logic: check `beat.visualType` to determine render mode:
  - `illustration` → `<img>` with Ken Burns animation (existing behavior)
  - `cinematic` → `<img>` with Ken Burns animation (from `cinematicUrl`)
  - `video` → `<video>` element, autoplay, muted, loop, 4s clip
- The effective image URL is: `beat.cinematicUrl ?? beat.imageUrl`
- Imagen 3 background layer drops to 0 opacity when ANY beat visual is active

### Shared Pydantic Types

```python
# visual_interleave_types.py
class BeatVisualAssignment(BaseModel):
    beat_index: int
    visual_type: Literal["illustration", "cinematic", "video"]
    generation_prompt: str | None = None  # null for illustration
```
